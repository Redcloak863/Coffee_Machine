This is my take on the Hyperskill Coffee Machine exerciss. Use of a class structure is probably overkill, but that was part of the project.

The "machine" is simple. It starts with an initial internal inventory of water, milk, coffee beans, and paper cups. It can "make" espresso, latte, or cappuccino. (The default is just a coffee. but this wasn't part of the project.) The user is presented with a menu to buy something, refill the machine, empty the cash box, or exit. If buy is chosen, the three drink options are shown. Not show is a "back" command that terminates the transaction. Not showing it is also part of the project's specifications. When a drink is selected, the program checks the existing inventory against the recipe for that item. If there are insufficient supplies, the first short item encountered is reported with an error message.
